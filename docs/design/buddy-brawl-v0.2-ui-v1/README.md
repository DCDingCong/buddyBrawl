# Buddy Brawl V0.2 UI 设计记录 V1

## 版本定位

本版本为 `Buddy Brawl V0.2 自动乱斗附游` 的第一版视觉方向归档。设计目标是把首层体验从资源管理和装备管理，调整为“打开看热闹、点一下打回去、武器表现推动成长”。

当前版本采用活泼的报纸/密卷视觉语言，保留轻量自动乱斗的节奏，同时避免复杂属性、战力、宝箱、金币升级和多装备槽造成的理解负担。

## 全局设计规则

- 所有用户可见文案默认使用中文。
- 不展示战力系统，只展示等级，例如 `胖达 Lv.6`。
- 不使用宝箱样式，不做宝箱页面。
- 金币不作为当前核心升级道具。
- 非弹窗页面底部统一三项导航：`首页`、`武器`、`好友排名`。
- 挑战和复仇只保留一个入口：有复仇状态时显示复仇图标，否则显示挑战图标。
- 主按钮使用竹绿色实心按钮；次级操作使用浅色按钮或文字按钮。
- 整体视觉采用米纸底色、竹绿色、墨色文字、少量琥珀成长提示和珊瑚红战斗强调。

## 页面说明

### 首页

首页是“刚才发生了什么”的事件中心，不是收益页。顶部展示熊猫形象和等级，主体展示 2 到 3 条自动事件摘要，例如练手、被偷袭、学会格挡。

收益不是退出后才开始挂机，而是在收取后重新开始计时。首页保留轻量计时条，显示类似 `下次记录 08:42`，旁边提供 `收取` 按钮。未满足收取条件时弱化按钮，不主动弹窗。

主操作为 `再打一场`，次入口为 `今日任务` 和 `看最新战报`。

### 对战动画页

对战动画单独成页，但动画不满屏。中上部使用轻量舞台卡片展示双方头像、动作线和伤害数字，顶部显示回合数，底部展示 1 到 2 条战斗日志。

操作以跳过、加速、继续为主，避免复杂控制。

### 战斗结算页

战斗结算页同时承担战斗报告的核心展示，不再另做一份完整报告页。视觉上使用“今日乱斗报”的报纸/密卷风格，包含胜负、关键回合、武器触发和战后成长。

示例信息：

- `板砖 Lv.2 挡住一拳`
- `板砖熟练度 +1`
- `胖达升到 Lv.7`

主操作为 `再打一场`，次操作为 `回首页`。

### 武器页

武器页只展示武器，不展示属性表、多装备槽或背包筛选。列表采用紧凑横向行：图标、武器名、等级格、右侧 `+`。

满足升级条件时 `+` 高亮可点；不满足条件时弱化或隐藏。点击具体武器后，在列表下方展开详情，不跳转新页面。

详情展示当前等级、下一级会在战报中出现的表现，以及成长条件，例如 `再触发 2 次`。

### 好友排名页

好友排名按等级排序，不按战力。前三名使用独立卡片表现，并给出称号或角标，例如 `竹林第一`、`乱斗新星`、`复仇对象`。

普通列表项保持紧凑，只展示头像、昵称、熊猫名、等级、最近互动状态和一个图标按钮。若该用户是复仇目标，按钮为复仇图标；否则为挑战图标。

### 报纸/密卷式战斗报告弹窗

该弹窗是可自定义组件，不是宝箱，也不是领奖弹窗。它用于登录后的自动事件总结、战斗结算摘要或从首页打开的最新战报。

结构为：

- 标题：例如 `胖达小报`
- 发生了什么
- 关键一击
- 武器表现
- 成长结果
- 操作：`知道了`、`去复仇`、`再打一场`
