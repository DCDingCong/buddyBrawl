# Buddy Brawl 产品设计 V0.2：自动化乱斗附游方向

## 1. 文档目标

本文档定义 `Buddy Brawl` 在 `V0.2` 方向上的产品调整。当前 `V0.1 Demo` 已经具备微信小程序、熊猫角色、挂机收益、装备、竞技和战报的最小闭环，但整体感知仍偏传统轻 RPG。

`V0.2` 的目标是把第一层体验从“管理装备、领取收益、强化属性”调整为“熊猫自动惹事、玩家看战报、点一下挑战或复仇、逐步获得装备与招式表现”。

核心方向：

`轻量化 + 自动化 + 社交战报 + 等级排行 + 装备招式成长`

## 2. 版本号规则

所有面向客户、测试用户或产品验收的版本号必须小于 `V1.0`，直到产品确认进入正式商业化阶段。

- 首个可体验 Demo：`V0.1`
- `V0.1` 的小迭代：`V0.1.1`、`V0.1.2`
- 方向性重构或主循环调整：`V0.2.1`
- 后续商业化前的大版本：`V0.3`、`V0.4`
- 不使用 `V1.0` 表达早期 Demo、内测版或首个开放测试版

当前三阶段首版应称为：`Buddy Brawl V0.1 Demo`。

本文档对应的开发方向可称为：`Buddy Brawl V0.2 自动乱斗附游`。

## 3. 版本定位

### 3.1 一句话描述

`Buddy Brawl` 是一款微信小程序自动乱斗附游。玩家拥有一只熊猫分身，熊猫会自动打架、惹事、练装备、触发招式；玩家回来后看发生了什么，并用一两个动作完成挑战、复仇或继续成长。

### 3.2 从 V0.1 到 V0.2 的关键变化

`V0.1` 更像：

`进入游戏 -> 领取挂机收益 -> 管理装备 -> 强化 -> 竞技 -> 看战报`

`V0.2` 应调整为：

`进入游戏 -> 看到熊猫刚刚发生的事 -> 点开战报 -> 挑战/复仇 -> 装备触发新招式 -> 自动变强`

区别不在于底层是否保留经验、金币、装备和成长，而在于玩家看到的第一层体验：

- 不先展示资源清单。
- 不先要求理解装备栏。
- 不先要求进入冒险页收菜。
- 首页先展示事件、战报、挑战、复仇和今日目标。

## 4. 设计原则

### 4.1 附游必须轻

轻不是没有系统，而是玩家第一分钟不需要理解系统。

首日玩家只需要理解三件事：

1. 我的熊猫会自动打架和惹事。
2. 战报会告诉我发生了什么。
3. 打架和升级会让装备、招式和熊猫成长。

### 4.2 附游必须自动

玩家不在线时，游戏仍要替玩家生成可读事件。

自动化不等于传统挂机收菜。传统挂机强调“离线多久，获得多少资源”；本项目应强调“你不在的时候，熊猫发生了什么事”。

### 4.3 成长必须服务战报

任何成长都要能在下一场战报中被玩家看见。

不推荐只给抽象数值：

`攻击 +3`

推荐把数值包装成战斗表现：

`板砖升到 Lv.2，以后有概率格挡对手一击。`

### 4.4 复杂系统后置

`V0.2` 不应在首页首屏暴露：

- 装备部位管理。
- 装备品质筛选。
- 复杂背包管理。
- 强化材料配方。
- 多层货币。
- 手动派遣冒险。

这些能力可以在底层数据结构保留，但不作为大众玩家的首层交互。

## 5. Q 宠大乐斗借鉴边界

Q 宠大乐斗的核心成功点不是传统 RPG 掉装，而是轻量社交 PK：

- 宠物是玩家的战斗分身。
- 战斗自动结算。
- 战报提供笑点、反转和传播内容。
- 好友挑战、复仇、排行制造回访压力。
- 武器、技能、帮派等复杂系统后期逐步展开。

本项目应借鉴结构，不照搬具体系统。

首页核心不是“资源总览”，而是：

`我的熊猫今天干了什么 -> 谁打我了/我打谁了 -> 现在可以打谁 -> 点一下开打`

## 6. 核心循环 V0.2

### 6.1 主循环

`回到小程序 -> 查看自动事件摘要 -> 打开战报 -> 挑战/复仇/再战 -> 装备或招式成长 -> 等待下一次事件`

### 6.2 首次体验循环

新玩家首次进入时，不应先进入复杂主页。

推荐流程：

1. 创建或获得熊猫。
2. 系统立即生成第一场教学乱斗。
3. 玩家看到短战报。
4. 战报中展示一次装备或招式触发。
5. 结算给一个基础装备或装备招式表现。
6. 玩家得到明确按钮：`再打一场`。

### 6.3 回访循环

玩家离开后再次进入时，不优先展示“领取收益”。

推荐展示：

`你不在的时候，胖达惹了 3 件事`

- `在竹林练手，板砖熟练度 +1`
- `被阿强的熊猫偷袭，已记录复仇目标`
- `偷偷观战，学会了锅盖格挡`

底部动作：

- `去复仇`
- `再打一场`
- `看战报`

## 7. 首页结构

首页从“资源状态总览”改成“事件和下一步动作中心”。

首屏展示优先级：

1. 熊猫形象和一句当前状态。
2. 最近 1 到 3 条自动事件。
3. 一个主按钮：`开打` 或 `去复仇`。
4. 次入口：`排行榜`。
5. 一个 `今日任务` 按钮。
6. 简化资源摘要，可弱展示金币，不展示复杂材料。

首页不应在首屏堆叠：

- 多个资源条。
- 复杂属性表。
- 装备栏。
- 长任务列表。

任务入口只保留一个按钮。点击 `今日任务` 后用弹窗展示每日任务和主线任务摘要。

## 8. 自动巡逻系统

### 8.1 系统定位

自动巡逻替代传统“挂机/冒险”首层体验。

旧概念：

`挂机收益`

新概念：

`熊猫离线自动惹事`

底层仍然可以按离线时间结算经验、金币和熟练度，但玩家看到的是事件流。

### 8.2 触发时机

`V0.2` 首版建议采用“登录或首页加载时结算”，不做后台定时任务。

结算入口：

- 用户手机号登录成功。
- 用户打开首页。
- 用户进入“刚才发生了什么”入口。

### 8.3 最大时间收益

自动巡逻必须有最大收益时间，避免无限离线收益。

建议首版配置：

- `maxSettlementMinutes`: 480，等于 8 小时。
- 离线超过 8 小时，收益只按 8 小时计算。
- 事件数量最多 5 条。
- 首页默认展示最多 3 条。
- 超出收益上限后可以生成一句提示文案，例如：`胖达后来睡着了，没有继续惹事。`

### 8.4 事件数量

推荐：

- 离线 10 分钟以内：0 到 1 条轻事件。
- 离线 10 分钟到 2 小时：1 到 2 条。
- 离线 2 小时到 8 小时：2 到 3 条。
- 离线 8 小时以上：最多 3 到 5 条。

不要按离线时长线性堆很多事件。

### 8.5 事件类型

- `practice`：练手事件，产出经验、金币或装备熟练度。
- `trouble`：惹事事件，制造轻战斗或战报入口。
- `ambushed`：被揍事件，制造复仇入口。
- `learned`：偷学事件，解锁装备招式表现。
- `found`：捡到事件，获得低阶装备或文案变体。
- `ranking`：排名事件，提示好友或等级排名变化。

### 8.6 奖励入账方式

`V0.2` 推荐自动入账。玩家不需要点 `领取收益`。

如果需要确认动作，按钮文案不叫 `领取`，而是：

- `知道了`
- `去复仇`
- `继续打`
- `看战报`

## 9. 装备与招式层级

`V0.2` 不删除装备，也不把招式做成完全独立的技能栏。

装备是商业化、成长和战斗表现的基础；招式是装备等级和熟练度在战报中的表现。

示例：

- `板砖 Lv.1`：敲一下。
- `板砖 Lv.2`：概率格挡。
- `板砖 Lv.3`：概率反击。
- `板砖 Lv.4`：暴击时触发额外连击或夸张文案。

后端应支持：

- 装备配置。
- 装备等级。
- 装备等级解锁招式。
- 战斗时根据装备等级自动推导可触发招式池。
- 重复获得或练习同一装备时提升熟练度或等级经验。

玩家不需要手动配置出战技能。

## 10. 金币与经济

金币保留，不删除。

金币定位：

- 后期商城货币。
- 购买基础装备。
- 购买低阶道具或基础外观。
- 可用于基础装备升级，但不作为首页主循环。

首层不建议展示：

- 强化材料。
- 多种碎片。
- 付费货币。
- 特殊活动币。

当前 `enhanceMaterial` 可以作为底层兼容字段保留，但 `V0.2` 首页和核心流程不应展示它。

## 11. 排行榜、好友和复仇

复仇入口不单独做成早期主入口，应合并进好友/排行榜玩法。

排行榜首版按等级排序，后续可叠加战力、胜场或赛季分。

列表项展示：

- 玩家昵称。
- 熊猫名称。
- 等级。
- 最近互动状态。
- 按钮：`挑战`。
- 如果此人最近打过我，按钮变为：`复仇`。

后端需要支持判断：

- 这个玩家是否可挑战。
- 这个玩家是否是复仇目标。
- 复仇是否已过期。
- 复仇战斗完成后是否清除或更新复仇状态。

## 12. 战报页

战报页是核心内容页，必须认真设计。

必须展示：

- 胜负。
- 关键回合。
- 触发的装备招式。
- 是否产生新装备、升级、熟练度或复仇机会。
- 下一步动作：`再打一场`、`去复仇`、`看装备`、`返回排行榜`。

战报承担教学：

`第 3 回合：胖达掏出板砖敲了一下，造成 18 点伤害。`

`板砖 Lv.2 触发：格挡了对手一次攻击。`

`战后成长：板砖熟练度 +1。`

## 13. 任务系统

每日任务保留，但必须简化。

每日任务：

1. 登录。
2. 查看战报。
3. 完成战斗，主动挑战或复仇都算。

主线任务单独做，围绕：

- 升级。
- 战斗次数。
- 领悟招式。
- 获得装备。
- 排名提升。

首页只展示一个 `今日任务` 按钮，打开后弹窗展示任务，不占首页首屏。

## 14. 后端数据建议

### 14.1 装备招式配置

```ts
interface EquipmentTechniqueRule {
  equipmentConfigId: string;
  requiredLevel: number;
  techniqueConfigId: string;
  triggerWeight: number;
}
```

### 14.2 招式配置

```ts
interface TechniqueConfig {
  id: string;
  name: string;
  type: "weapon_action" | "guard" | "counter" | "passive" | "special";
  rarity: "common" | "uncommon" | "rare" | "epic";
  triggerCondition: "always" | "on_attack" | "on_defend" | "on_critical" | "low_hp";
  baseWeight: number;
  cooldownRounds: number;
  effect: {
    kind: "damage" | "block" | "counter" | "buff";
    value: number;
  };
  reportTextTemplates: string[];
}
```

### 14.3 玩家装备进度

现有 `EquipmentInstance` 可继续使用，但需要补充或复用：

- `enhanceLevel` 作为装备等级。
- 新增 `proficiency` 或独立 `PlayerEquipmentProgress`。
- 新增最近触发时间和触发次数可选字段。

### 14.4 自动巡逻事件记录

```ts
interface PatrolEvent {
  id: string;
  playerId: string;
  type: "practice" | "trouble" | "ambushed" | "learned" | "found" | "ranking";
  summary: string;
  rewards: Reward[];
  relatedBattleId?: string;
  relatedPlayerId?: string;
  relatedEquipmentConfigId?: string;
  relatedTechniqueConfigId?: string;
  actionType?: "revenge" | "battle" | "viewTechnique" | "viewReport";
  createdAt: string;
  seenAt?: string;
}
```

### 14.5 自动巡逻状态

```ts
interface PatrolState {
  playerId: string;
  lastSettledAt: string;
  dailyGeneratedCount: number;
  pendingEventCount: number;
}
```

## 15. V0.2 首版必须做

- 版本号与文档统一到 `V0.x`。
- 首页自动事件摘要。
- 自动巡逻结算与最大收益时间。
- 装备等级触发招式配置。
- 战斗中随机触发装备招式。
- 战报展示招式触发过程。
- 战后装备熟练度或成长结算。
- 好友/等级排行榜中的挑战和复仇按钮所需后端字段。
- 每日任务弹窗数据。
- 主线任务基础数据。

## 16. V0.2 首版不做

- 复杂背包。
- 装备品质筛选。
- 多地图关卡。
- 复杂碎片合成。
- 多宠物阵容。
- 完整赛季。
- 付费强化。
- 金币商城正式开放。

## 17. 验收标准

`V0.2` 产品体验完成时，应满足：

1. 新玩家 1 分钟内能完成第一场自动乱斗。
2. 第一场后能看到装备或招式表现。
3. 玩家不需要手动配置技能也能理解自己变强了。
4. 玩家离线后再次进入，能看到自动生成的事件摘要。
5. 自动巡逻收益存在最大时间上限。
6. 自动事件至少有一类能引导玩家去复仇或再战。
7. 战报能解释装备招式如何触发。
8. 首页首屏不会展示复杂属性表或资源清单。
9. 每日任务只包含登录、查看战报、完成战斗。
10. 排行榜能按等级展示用户，并提供挑战/复仇按钮所需状态。

## 18. 推荐迭代顺序

1. 统一版本号和文档命名。
2. 定义装备招式配置。
3. 战斗引擎接入装备等级触发招式。
4. 战报展示装备招式触发。
5. 增加战后装备熟练度或成长结算。
6. 增加自动巡逻事件生成和最大收益时间。
7. 首页改为事件摘要和主动作。
8. 排行榜改为等级排名，并提供挑战/复仇状态。
9. 每日任务和主线任务拆分。
10. 根据测试反馈再决定是否保留深层强化入口。

## 19. 产品判断

`V0.2` 的关键不是删掉成长系统，而是改变成长系统的外观。

金币、装备、等级和任务都可以保留，但它们不应压在首页首屏。玩家应该觉得：

`我的熊猫越来越会打架，也越来越会惹事。`

最终体验应接近：

`打开 -> 看热闹 -> 打回去 -> 装备触发新花样 -> 退出后继续发生故事`
